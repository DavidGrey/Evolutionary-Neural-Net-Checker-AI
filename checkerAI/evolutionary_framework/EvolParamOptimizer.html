Class EvolParamOptimizer

Class EvolParamOptimizer

All Implemented Iterfaces:
           Evol_Object

public class EvolParamOptimizer

This class optimizes the evolutionary search parameters population size and number of elitist elements for the Evol_Algorithm class for a specific instance of the Evol_Object interface. To use, construct one instance of this class and evolve it using Evol_Algorithm as you would with any other Evol_Object.

How it works: The initial population is generated randomly with population sizes of 1 to 500. Number of elitist elements is generated randomly from 0 to population size. A mutation is more or less defined as: 25% chance num elitist decrements by 1, 25% chance num elitist increments by 1, 25% chance population size increments from 0 to 4 (randomly), 25% population size decrements from 0 to 4 (randomly). Checks ensure that population size > 1, num elitist < population size, and num elitist >= 0. Fitness is calculated by performing the evolutionary search N times (default 10) and averaging the results.

Constructor detail

EvolParamOptimizer

public EvolParamOptimizer(Evol_Object rt, int stop_val, int max_generations, boolean optimize_for_fitness, int max_solut)

           Creates an instance of EvolParamOptimizer, which can then be evolved using Evol_Algorithm. EvolParamOptimizer determines what the best population size and number of elitist elements are for evolving a specific instance of Evol_Object with Evol_Algorithm.

           Parameters

rt - An instance of an Evol_Object which contains a representation of the problem that you wish to solve.
stop_val - Stop value for rt. See Evol_Algorithm.
max_generations - Number of generations to evolve (if optimization problem) or number of generations before failure (if looking for a specific solution).
optimize_for_fitness - If this is true, the EvolParamOptimizer object's fitness will be based on rt's fitness after max_generations. If this is false, the fitness will be based on resources used to find the solution (optimizes for finding the solution faster). With the latter, be sure to make max_generations fairly high because a failure will be measured the same as a success, population size * number of generations. By making max_generations higher, you are penalizing objects which fail to find a solution. For example, use true for the TSP and false for the SAT.
max_solut - Before calculating its fitness, EvolParamOptimizer calls max_num_solutions(max_solut) on internal Evol_Algorithm object. -1 disables this option.

Method detail

print_params

public void print_params()
Prints the population_size and number of elitist elements for this instance of EvolParamOptimizer. You can use this to print the parameters of the returned EvolParamOptimizer from your call to Evol_Algorithm's search() method.

set_trials

public void set_trials(int num_trials)
Because an evolutionary search relies on randomness, each trial run will give a different result. To compute the fitness for a set of parameters, each parameter set (EvolParamOptimizer object) is tested a number of times (default 10) and averaged. This function sets the number of trials.
